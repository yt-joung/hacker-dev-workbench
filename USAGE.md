# 🛠️ HackerDev Workbench 사용 가이드 (Advanced Pentesting Guide)

HackerDev Workbench는 웹 애플리케이션 보안 진단 및 리버싱을 위한 전문적인 고성능 작업 환경을 제공합니다. 본 가이드는 2단계 네비게이션 구조로 개편된 각 기능의 상세 사용법을 설명합니다.

---

## 🧭 인터페이스 구조 (2-Tier Navigation)

화면 상단의 **메인 카테고리**를 선택하면, 해당 카테고리에 속한 **서브 탭**들이 나타납니다.

| 메인 카테고리 | 용도 | 포함된 기능 |
| :--- | :--- | :--- |
| **⚡ SCAN** | 정적 분석 및 환경 정찰 | RECON, FORMS, JSVIEW, INSPECT |
| **🌐 NET** | 네트워크 트래픽 및 세션 분석 | NET LOG, REPEAT, JWT |
| **⚔️ ATTACK** | 동적 조작 및 공격 도구 | DEV, STORE, LIVE |
| **🛡️ REPORT** | 진단 결과 및 자산 관리 | VULNS, HISTORY, LIBRARY |

---

## ⚡ [SCAN] 정적 분석 및 환경 정찰

### 1. RECON (DOM 정찰)
- **Visual Toggle**: 페이지 내 숨겨진 요소(`hidden`, `display:none`)를 강제로 노출시키고 테두리를 표시합니다.
- **추출 목록**: 원클릭으로 요소를 선택하여 `DEV(Workbench)`로 코드를 전송해 즉시 조작할 수 있습니다.

### 2. FORMS (폼 분석)
- 페이지 내 모든 `<form>` 요소를 탐색하여 **CSRF 토큰 누락, GET 메소드 사용, 파일 업로드 필드** 등 보안 미흡 사항을 자동으로 식별합니다.

### 3. JSVIEW (JS 소스 뷰어)
- 로드된 자바스크립트 소스를 가독성 있게 포맷팅(Beautify)합니다.
- **자동 스캔**: `token`, `key`, `eval`, `innerHTML` 등 20여 개의 민감 키워드를 위치(Line Number)와 함께 자동 탐색합니다.

### 4. INSPECT (프레임워크 분석)
- **Vue.js (Pinia/Vuex)** 및 **React (Fiber)**의 내부 상태(State)를 추출합니다.
- 메모리상에 존재하는 민감한 전역 변수나 사용자 정보를 탈취하여 진단 근거로 활용할 수 있습니다.

---

## 🌐 [NET] 네트워크 트래픽 및 세션 분석

### 1. NET LOG (트래픽 모니터링)
- 현재 페이지에서 발생하는 API 호출 등 모든 통신 기록을 캡처합니다.
- 로그를 클릭하면 **REPEAT** 또는 **JSVIEW** 탭으로 즉시 전송하여 분석할 수 있습니다.

### 2. REPEAT (Request Repeater)
- 요청을 수정하여 재전송하는 도구입니다.
- **CORS 우회 & 세션 유지**: 페이지 컨텍스트(`MAIN world`)에서 실행되므로, 실제 사용자의 인증 쿠키가 자동으로 포함된 상태로 요청을 보낼 수 있습니다.

### 3. JWT (JWT 인스펙터)
- 캡처된 JWT 토큰을 실시간으로 디코딩합니다.
- **공격 페이로드 생성**:
  - `alg:none`: 알고리즘 검증 미흡 취약점 테스트.
  - `exp 연장`: 세션 만료 시간 강제 조작 테스트.
  - `권한 상승`: Role 정보(admin 등) 조작 테스트.

## ⚔️ [ATTACK] 동적 조작 및 공격 도구

### 1. DEV (Workbench)
- **자바스크립트 인젝션**: 라이브 스크립트 에디터를 통해 실시간으로 코드를 주입하고 실행 결과를 전용 콘솔에서 확인합니다.
- **인코더/디코더**: Base64, URL, Hex, HTML Entity 등 진단 시 필요한 변환 도구를 제공합니다.

### 2. STORE (저장소 편집기)
- **LocalStorage / SessionStorage / Cookie**를 실시간으로 열람하고 값을 강제로 수정합니다.
- 인증 정보 변조를 통한 권한 우회 진단 시 필수적으로 사용됩니다.

### 3. LIVE (런타임 이벤트)
- `postMessage`, `eval`, `setTimeout` 등 런타임에 발생하는 동적 이벤트를 후킹(Hooking)하여 데이터 흐름을 추적합니다.

---

## 🛡️ [REPORT] 진단 결과 및 자산 관리

### 1. VULNS (취약점 목록)
- 진단 과정에서 자동으로 식별되거나 수동으로 등록한 취약점 리스트입니다.
- **상세 데이터**: 발견 위치, 증거(Evidence), 위협 설명, 조치 권고 사항을 포함합니다.

### 2. LIBRARY (페이로드 무기고)
- 자주 사용되는 자바스크립트 페이로드를 관리합니다.
- **커스터마이징**: 페이로드의 추가, 수정, 삭제가 가능하며 브라우저에 영구 저장됩니다.
- **파일 관리**: `Export` 기능을 통해 자신의 무기고를 JSON 파일로 백업하고, `Import` 기능을 통해 다른 환경에서 로드할 수 있습니다.

### 3. HISTORY (진단 기록)
- 이전 사이트에서 수행했던 스캔 데이터와 발견된 항목들을 저장하고 다시 불러올 수 있습니다.
